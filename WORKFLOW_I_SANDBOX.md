## I0. 模式定位（与 G / 查证模式的区别）

| 维度 | 查证模式（A~F） | G 跑模组 | **I 沙盒** |
|---|---|---|---|
| 用户身份 | 提问者 | 玩家 | 玩家 |
| AI 身份 | 资料员 | DM（按模组剧本带剧情） | DM（开放世界 + 即兴生成） |
| 推进方式 | 一问一答 | 章节 / boxed text 触发 | 玩家声明 + downtime 行动 |
| 时间尺度 | — | round / scene / chapter | hour / day / week / month / season / year |
| 主线 | — | 模组预设 | 玩家自驱（短/中/长期野望） |
| 触发节点纪律 | — | 严格 G2.5（模组遭遇必触发） | 转为「未结事件清单」（已设钩子按期到期） |
| DC / 遭遇 | — | 模组预设 | DM 即兴 + DMG24 表生成 |
| 关键资源 | `docs/extracted/` | `docs/modules/<模组>/` | `campaigns/<战役>/sandbox/` + 设定书 |

### 🚫 核心原则（与 G 共享 + 沙盒专属补充）

继承 G 工作流的全部信息揭示边界（G4：保密 / 编号不外泄 / DC 不外泄 / 模组元数据不外泄 / 选项菜单零 meta），与纯散文输出风格（G2 末段）。**额外强调三点**：

**1. 风味是叠加层，不是替换层**（与 G2.5 同源原则）：
- 沙盒**不等于**"取消战斗的休闲游戏"
- 接的委托可能死人；领地受袭可能 TPK；生意失败可能破产；婚姻可能背叛
- 经济压力 / 时间压力 / 关系成本是沙盒的张力来源，**不能因为"日常向"就架空**
- 反例：玩家接"护送商队"委托 → 沿途必须有真实遭遇 / 检定 / 风险，不能仅描述 RP 后直接给报酬

**2. 玩家自驱不等于 DM 失踪**：
- 沙盒**最易翻车的姿态**是 DM 变成"yes-man"：玩家说啥都给糖
- DM 必须维护「未结事件清单」（钩子 / 债务 / 承诺 / 季节性 / 政治）—— 这些事件按 in-game 时间到期，**玩家不能用"快进"逃避**
- 玩家**没有目标**时（沙盒最常见的"卡壳"），DM 通过 NPC / 环境 / 委托池主动给钩子，但**不强制**

**3. 世界状态一致性是命门**：
- 三个月前的酒馆老板今天必须叫同名字
- 上次借的钱本月底必须有人来收
- 买的房子要写进 `assets/properties.md`，下次进入时仍在
- **存档比叙事更重要**——沙盒翻车 90% 是因为状态档维护跟不上

### 🚫 沙盒不主动 OOC（与 G 一致）
in-character 优先，玩家明确出戏才回答 OOC。即兴生成的 NPC / 地点 / 事件要在叙事**当下**就写入对应档案，不能等"以后再补"——否则下次进入时这些 NPC 会"消失"。

## I1. 开桌前准备

### I1.1 沙盒五问（首次开桌时一次性问完）

1. **起始地**：MVP 推荐**博德之门**（剑湾 / 费伦设定）。其他可选：
   - 深水城（费伦最大都市，机会最多）
   - 烛堡（学者沙盒）
   - 候门城（北地边境，硬核）
   - 自创小镇（如玩家想从乡村起步）
   - 引用：[剑湾冒险者指南/欢迎来到国度/托瑞尔诸地.md](../../docs/extracted/剑湾冒险者指南/欢迎来到国度/托瑞尔诸地.md) 选地点
2. **DM 风格**：同 G1 步骤 2，让玩家从 [profiles/dm-styles/](profiles/dm-styles/) 选预设
3. **房规 + dice_mode**：同 G1 步骤 3
4. **角色**：等级 + 起始财富（按 DMG24 ["不同等级的冒险情景"](../../docs/extracted/城主指南2024/4.创建冒险/布置背景/不同等级的冒险情景.md)）。若无卡走工作流 B
5. **玩家野望（关键）**：问玩家
   - **短期野望**（1-3 个月可达成）："我想攒钱买一间小铺子"、"我想加入冒险者公会"
   - **中期野望**（半年-1 年）："我想娶到艾莉安娜"、"我想在城里开一家魔法用品店"
   - **长期野望**（多年）："我想在剑湾买下一座要塞"、"我想成为博德之门的议长"

野望写进 `dm-only/sandbox-notes.md` 的"野望追踪"段，作为 DM 调遣钩子 / 委托 / NPC 的依据。**野望不告诉玩家档案的存在**（保密），但 DM 据此往世界里塞相关线索。

### I1.4 初始化关键档案

开桌前必须 Write 出以下文件（用模板，详见 §I10）：

0. **🎚️ `.fathom-panels/dnd5e/threads/<threadId>/session.json`** —— **面板的命根子，第一件就写**（`mode=I` + campaign + players + sandbox 初值；threadId 取自 `<workspace>/.fathom-context.json`，无该文件的 Claude Code 环境才跳过）。**不写它，面板永远停在"未开桌"**——玩家一眼可见。Schema 与状态行见 [SKILL.md §session.json / §状态行](SKILL.md)。
1. **`sandbox/calendar.md`** — 历法 + 当前日期 + 已知未来事件（节日 / 季节性事件 / 玩家已知截止日）
2. **`sandbox/locations/<起始地>.md`** — 起始地的详细档案（街区 / 主要建筑 / 已知 NPC 名单 / 物价水准 / 治安）
3. **`sandbox/downtime-menu.md`** — 当前角色可用的 downtime 行动（按角色技能 / 阶级 / 现金过滤）
4. **`dm-only/sandbox-notes.md`** — 野望追踪 + 伏笔池

其他档案（commissions / relationships / assets / family）按需懒加载——玩家第一次涉及到时再创建。

### I1.5 开场叙事

按 DM 风格写一段开场，把角色"放置"到起始地。**不**铺剧情、**不**给任务，只给环境感与可探索的提示线索：

> 你站在博德之门的下城码头。脚下湿漉漉的木板还带着昨夜大雨的痕迹，海风裹挟着腌鱼和柴油的气味。远处大公爵广场的钟楼传来三下闷响——傍晚了。你刚从克利里克船长的"海鸥号"下来，口袋里 50 金币，背包里几件冒险者的家伙。
>
> 现在去哪里，做什么——由你决定。

然后等玩家声明。**不给选项菜单**（沙盒第一拍最容易破坏的就是"开放感"）。

## I2. 日常节奏（每个 in-game 日的循环）

### I2.1 一日四段

每个 in-game 日默认划分为 4 段，玩家可在每段声明做什么：

| 时段 | 时间 | 典型行动 |
|---|---|---|
| 晨 | 6-12 时 | 工作 / 训练 / 市场 / 拜访 |
| 午 | 12-18 时 | 工作续 / 跑腿 / 接委托 / 旅行 |
| 晚 | 18-24 时 | 酒馆 / 社交 / RP / 委托夜间段 |
| 夜 | 0-6 时 | 长休 / 夜行 / 偷窃 / 守夜 |

**不强制**玩家声明 4 段都做啥——玩家可以说"我整天都在图书馆研究"（一日只算一个 downtime 行动），也可以"早上市场，下午找艾莉安娜，晚上酒馆"。

### I2.2 推进循环（每段重复）

每段玩家声明 → DM 推进 → 触发事件 → 结果叙事 → **状态有变化则先写 session.json、再在回复结尾发一行 `〔状态〕` 状态行**（见 [SKILL.md §状态行](SKILL.md)；HP/位置/时间/沙盒态任一变化即必写必发） → 下一段。

1. **玩家声明本段做什么**
2. **DM 判定**：
   - 是 downtime 行动 → 走 §I4 行动菜单
   - 是社交 / RP → 走 §I6 关系演进
   - 是接委托 / 战斗 → 切到 G2 节奏（沙盒**内嵌**G 工作流）
   - 是经营资产 → 走 §I7
   - 是"快进 X 天/周/月" → 走 §I3 时间推进协议
3. **触发即兴事件**（约 20% 概率，按地点危险等级调整）：
   - 街角小偷 / 路边乞丐 / 偶遇旧识 / 突发治安事件 / 见到稀奇人物 …
   - 这些事件**不抢戏**，作为环境质感（玩家可选择介入）
4. **结果叙事 + 推进时段**：散文风格，纯文本无 markdown（同 G2）
5. **检查 sandbox/calendar.md 未结事件**：本段是否触发？触发则插入

### I2.3 何时切到 G 模式（战斗 / 重要遭遇）

玩家接的委托进入"动作场景"（战斗 / 紧张潜行 / 高 stakes 检定串）时**完整切到 G2 节奏 + G3 战斗管理**。具体触发：

- 玩家说"我接了这个委托，出发"——把场景切到目标地点，按 G2 描述场景 + 等声明
- 委托遭遇/战斗：用 G3 战斗管理（写 `combat/active.md`）
- 重要 NPC 对话 / 关键决策点：保持 G2 in-character 严谨
- 委托完成回城 → 切回 §I2 日常节奏

简言之：**沙盒是上层框架，G 是动作场景的执行器**。两者无缝切换，玩家不需要切 mode 命令。

### I2.4 跑团叙事风格

完全继承 G2 末段「纯散文，无 markdown」原则：

- NPC 台词无加粗 / 斜体
- 场景描述用连贯散文，不列 bullet
- 选项菜单零 meta（不带 DC / 不预告内容 / 不暴露事件触发条件）
- 例外：玩家 OOC 问规则时可用 markdown 列要点

## I3. 时间推进协议

### I3.1 五档推进

| 档位 | 触发 | 玩家声明示例 |
|---|---|---|
| 段推进 | 默认每完成一段行动 | 自动 |
| 日推进 | 玩家"睡到第二天" / 完成最后一段 | 自动 |
| 周推进 | 玩家明确"花一周做 X" | "我接下来一周都在图书馆研究" |
| 月推进 | 玩家明确"花一个月做 X" | "我花一个月训练剑术" |
| 季 / 年推进 | 玩家"我安心住下来等孩子出生" | "快进到春天" / "跳到孩子 5 岁" |

### I3.2 结算清单（每次快进时按档位执行）

**所有快进都触发以下结算**（清单按顺序走，不要漏）：

#### 1. 资产收支（每月一次结算，跨月推进时累加月数）

读 `sandbox/assets/treasury.md` 和 `properties.md` `business.md`：

- 房产月维护费（10% 购入价 / 12 ≈ 简单估算）
- 产业月利润（按业务规则，详见 §I7）
- 雇员工资
- 生活水准固定开销（按 PHB24 "生活水准"段）

Write 进 `treasury.md` 末尾"月度流水"段，并 Edit 角色卡 wealth 字段 + `players/<名>.md` 同步。

**若现金<0**：DM 必须**事件化**告知玩家——债主上门 / 雇员闹辞职 / 房东催租，**不是**直接负数。

#### 2. 关系衰减（每月一次）

读 `sandbox/relationships/*.md`：

- 本月未互动的 NPC：好感度 -5（同居 / 配偶 / 直接亲属除外）
- 本月有"礼物 / 共冒险 / 重要事件"的 NPC：好感度按事件加值
- 跨越阶段阈值时触发"阶段转移事件"（见 §I6）

#### 3. 未结事件检查（**最关键**）

读 `sandbox/calendar.md` 的"未结事件清单"。任何到期事件必须在该推进段内触发：

- 债务到期 → 债主上门
- 委托截止日 → 失败后果 / 客户找上门
- 政治事件 / 议会决议 → 在 world-state.md 反映
- NPC 自身的承诺 / 行动（如"塔拉克说一个月后给你写信"）→ 写信到达
- 季节事件（节日 / 税期 / 商队来访）→ 触发对应活动
- 怀孕里程碑（如适用）→ 见 §I8

**绝不可跳过到期事件**。若玩家试图用快进"跳过"已知钩子，DM 必须在快进过程中将事件以最自然的方式插入，并停下让玩家应对。

#### 4. 季节事件 / 节日

读 `sandbox/calendar.md` 的"季节事件"。费伦历法见 [剑湾冒险者指南/欢迎来到国度/国度的历法.md](../../docs/extracted/剑湾冒险者指南/欢迎来到国度/国度的历法.md)：

- 春分 / 夏至 / 秋分 / 冬至 → 各地不同庆典
- 月底 / 季末 → 税期 / 商队 / 法庭日
- 玩家曾参加的节日 → 同 NPC 关系自动 +2（共度时光）

#### 5. 孕期 / 孩子成长（若 family.md 有相关条目）

见 §I8 详细规则。

#### 6. 角色状态重置

跨日推进自动长休（HP 满 / 法术位回满 / 资源回满）。提醒玩家选择新备战法术（若是法师 / 牧师 / 圣武士 / 奇械师）。

#### 7. 角色卡 ↔ .md 同步

每次快进结束**询问玩家**："期间你想在角色卡 HTML 上调整什么吗？（消耗的药水 / 学的法术 / 买的装备等）"——同 G7 步骤 10。

### I3.3 快进的"事件密度"控制

跨周以上的快进，DM 不能"压缩到一行"——必须给玩家**至少 1-2 个该期间的场景节拍**（哪怕只是一句叙事）：

- 一周：1-2 个节拍（某天去市场买了 X，遇到 Y）
- 一月：3-5 个节拍 + 月底结算
- 一季：每月 1-2 个节拍 + 季节事件
- 一年：每季 1 个事件高潮 + 年底总结

节拍由 DM 从「未结事件清单」+「即兴生成」混合选取。例：

> 你在图书馆度过了平静的一个月。第二周遇到一位老法师询问你查的书；第三周收到塔拉克的信，说海藻洞穴又有新动静；月底你结清了房租，发现"灯塔之眼"的店主最近脸色越来越差……

### I3.4 玩家试图"过度快进"时

玩家说"我跳到 10 年后"这种**极端快进**时，DM 应：

1. 先警告："这段时间会发生很多事，包括 [列出 1-2 个已知大事件]。你确定要这样压缩吗？"
2. 若玩家坚持，按"年推进 × N"逐年走，每年至少给 1 个节拍 + 1 个未结事件触发
3. 极端情况（如玩家想跳过即将到期的危险事件）DM 应让该事件**在快进过程中触发**并停下

- PHB24 + Xanathar's Guide（资料库 `珊娜萨的万事指南`）

**MVP 必做的 6 个核心行动**：

#### 1. 接委托 → 走 §I5
玩家声明"我去看公会布告 / 找 X 接活" → 切 §I5 委托流程。

#### 2. 训练 / 学习
- **学新工具熟练**：250 GP + 250 天（PHB24，找师傅可加速）
- **学新语言**：同上
- **获得新专长**：DMG24 允许的"训练获得专长"选项（house-rule，默认关闭，需玩家开桌时声明）
- **法术研究**（仅施法者）：抄写卷轴成法术书 / 创造新法术（DMG24 ["创作法术"](../../docs/extracted/城主指南2024/3.地下城主工具箱/创作法术.md)）

DM 操作：Edit `players/<名>.md` 的"训练进度"段记录已投入天数；完成时同步角色卡 HTML（提醒玩家）。

#### 3. 经商 / 管理产业
玩家有产业（见 §I7）时，每月触发一次经营事件：
- 顺利月：营业额 = 基础 × (0.8~1.2)
- 突发：员工纠纷 / 客户投诉 / 竞争对手 / 市场波动
- 每月让玩家做一次相关检定（说服 / 洞察 / 历史 / 调查）

#### 4. 社交 / 拜访 NPC → 走 §I6
玩家说"我去找艾莉安娜吃晚餐" → 进入 §I6 关系演进流程。

#### 5. 造物
- **炼金**（药水 / 毒药）：DMG24 ["创作魔法物品"](../../docs/extracted/城主指南2024/3.地下城主工具箱/创作魔法物品.md)
- **抄写卷轴**（仅施法者）：法术等级 × 50 GP + 1 天/环
- **铭文 / 法器**（高级，需对应专长 / 物品）：见 DMG24

简化规则：让玩家声明造物 + 投智力（炼金术工具）→ DC 由物品稀有度决定 → 成功消耗材料 + 时间 → 失败浪费一半材料。

#### 6. 工作（短期赚钱）

参考 PHB24 生活水准。无产业的玩家可短期打工：
- 低技能（搬运 / 跑堂）：1 GP / 日
- 中技能（书记员 / 学徒）：2 GP / 日 + 技能加值 GP
- 高技能（接私活 / 表演）：5+ GP / 日 + 检定（说服 / 表演 / 工具）

### I4.2 进阶行动（第二阶段加入）

#### 7. 研究 / 信息搜集
玩家想查某个主题（"火花塞是谁" / "这件装备的来历" / "幽暗地域的入口"）：
- 选地点：图书馆 / 行会 / 酒馆传闻 / 神殿档案
- 投智力（调查） / 智力（历史） / 魅力（说服）
- DC 由信息稀有度决定（10 ~ 25）
- 时间消耗：1-7 天
- 结果：成功 → DM 透露信息（按 G4 信息揭示边界，可分档透露）

#### 8. 休养 / 治疗
长期疾病 / 诅咒 / 心理创伤的恢复：
- 普通病：1-3 周 + 体豁
- 诅咒：参考 [DMG24/3.地下城主工具箱/诅咒与魔法疫病.md](../../docs/extracted/城主指南2024/3.地下城主工具箱/诅咒与魔法疫病.md)
- 心理创伤（DMG24 [恐惧与精神压力](../../docs/extracted/城主指南2024/3.地下城主工具箱/恐惧与精神压力.md)）：神职咨询 / 朋友陪伴 / 时间

#### 9. 灰色 downtime（house-rule 默认关闭）
- 赌博：投魅力（欺瞒） vs DC 15 → 赢 1d10 GP / 输 1d10 GP
- 走私 / 偷窃：高风险高收益（DMG24 阴谋 + 治安）
- 间谍 / 情报：为赞助者收集情报

#### 10. 神祇祈祷 / 寻求指引
玩家信仰神祇时：
- 庙宇祭品（金币 / 物品） → 投智力（宗教）/魅力（说服）
- 成功 → 获得占卜（DC 由问题复杂度决定）或暂时祝福
- 见 [剑湾冒险者指南/欢迎来到国度/费伦诸神详情.md](../../docs/extracted/剑湾冒险者指南/欢迎来到国度/费伦诸神详情.md)

### I4.3 行动选择呈现给玩家

**不要**主动列菜单给玩家——零 meta 原则。玩家声明意图（"我去酒馆打听消息"、"我想学锁匠工具"）→ DM in-fiction 描述场景 + 等检定声明。

只有玩家明确出戏问"我可以做什么"时，才用 markdown 列出可用行动清单（OOC 模式）。

## I5. 委托系统

### I5.1 委托池来源

读 `sandbox/commissions/active.md` 维护当前可接委托。来源：

1. **冒险者公会 / 雇佣兵公会**布告板（最普遍）
2. **赞助者**派单（玩家加入了某个组织——[塔莎/团队赞助者/](../../docs/extracted/塔莎的万事坩埚/团队赞助者/)）
3. **熟人请托**（NPC 朋友 / 邻居 / 亲属求助）
4. **偶遇事件**（街上撞见 / 酒馆耳闻）
5. **政府 / 派系悬赏**（声望系统升级后解锁——DMG24 [声望.md](../../docs/extracted/城主指南2024/3.地下城主工具箱/声望.md)）

### I5.2 委托档案 schema

每个委托一个条目（写在 `commissions/active.md` 里，完成后归档到 `completed/NNN_<名>.md`）：

```yaml
---
id: COM-001
title: 寻找走失的猫
issuer: 老妇人玛丽埃塔（下城）
type: 寻物                            # 杀怪/护送/调查/偷窃/外交/拯救/寻物/谋杀；演化后可写 "寻物 → 调查"
posted_date: 风岚月 第 3 日
deadline: 风岚月 第 10 日
reward: 10 GP + 玛丽埃塔的好感        # 简单情况：单值字符串；演化后改成 {base, pivot} 对象（见 §I5.7）
difficulty: easy                      # easy/medium/hard/deadly（按 PC 当前等级）
risk: low                             # 风险评估（low/medium/high/extreme）
scenes: 1                             # 预估场景数
status: open                          # open/accepted/in_progress/evolved/completed/failed/expired
chain:                                # 任务链关系（默认 null，无父子）；演化时由 DM 填写
  parent: null                        #   父任务 id
  spawned: []                         #   由本任务派生的子任务 id 列表
revealed_branches: []                 # 已揭示的分支记录（玩家行动触发后由 DM 追加；见 §I5.7）
secret: |
  🔒 这只猫被一个练习召唤术的法师学徒偷去做实验。
  找到猫窝在贫民窟某处，学徒会试图说服玩家放过他（DC 14 说服）。
  🔒 潜在演化分支（可选预埋）：若玩家审讯学徒 + DC 12 调查通过 → 揭示其导师在编纂禁咒书 → 派生新委托
---

## 公开描述
"我可怜的小白……出门已经三天了，从来不会这样的。请帮我找找它吧。"

老妇人住在下城西区，那只白猫戴着银色项圈。她愿意付 10 金币。

## 演进日志
（追加式；演化触发时由 DM 填写 —— 日期 / 触发条件 / 揭示内容 / 派生 COM-XXX。详见 §I5.7）
```

**关键字段说明**：

- `status` 新增 `evolved` 值：原目标已完成但事件升级，主线已派生到子委托。区别于 `completed`（彻底结案）。
- `chain.parent` / `chain.spawned`：建立任务链结构，归档后仍可追溯"COM-005 是由 COM-001 衍生的"。
- `reward` 双形态灵活：日常委托写单字符串即可；演化后改写成对象 `{base: "10 GP + 玛丽埃塔好感", pivot: "100 GP + 焰拳好感（挂在 COM-005）"}` 分段记录。
- `revealed_branches` vs `secret`：`secret` 是 DM **预埋**的潜在剧情（玩家不可见）；`revealed_branches` 是**实际揭示**给玩家后的记录（DM 写入时机 = 玩家行动触发后）。
- `## 演进日志` 段：散文式追加，纯文本即可，作为桌末复盘的"剧情转折流水"。

### I5.3 委托执行流程

1. **接受**：玩家声明接 → Edit 委托 status = accepted + 当前日期
2. **侦察 / 调查**：玩家声明做啥（问 NPC / 调查现场 / 跟踪 / ...）→ 切 G2 节奏跑场景
3. **核心场景**（按 difficulty 配比）：1-3 个 scene，按 G3 战斗管理跑战斗
3.5. **演化判定**（核心场景中或结束时检查；详见 §I5.7）：
   - 玩家是否通过审讯 / 调查 / 检定揭示新线索 / 更大阴谋？
   - 玩家的行动是否暴露 `secret` 段预埋的"潜在演化分支"？
   - 命中则当场处理：在原档案 `## 演进日志` 段追加一行 + 在 `active.md` 新建子委托 + 回填 `chain` 字段
4. **完成 / 失败**：
   - 完成 → 报酬入账（Edit `treasury.md` + 角色卡 wealth）+ 发布者好感 +10（Edit `relationships/<名>.md`）+ 声望加值 + 归档
   - 失败 → 发布者好感 -5 + 可能后续负面（如被列入黑名单）
   - 超期未完成 → 自动 expired + 发布者好感 -10
   - **演化型完成**（status: evolved）→ 发 `reward.base` + 归档原委托 + 主线交接给子委托继续推进（详见 §I5.7）

### I5.4 委托难度与报酬基准（按 PC 等级）

| PC 等级 | easy 报酬 | medium 报酬 | hard 报酬 | deadly 报酬 |
|---|---|---|---|---|
| 1-4 | 5-20 GP | 25-100 GP | 100-500 GP | 500+ GP |
| 5-10 | 50-200 GP | 200-1000 GP | 1000-5000 GP | 5000+ GP |
| 11-16 | 500-2000 GP | 2000-10000 GP | 10000+ GP | 极珍稀法宝 |
| 17-20 | 5000+ GP | 50000+ GP | 传奇法宝 | 神器 / 政治回报 |

报酬非金钱时（信息 / 法宝 / 声望 / 人情）按 DMG24 [冒险奖励](../../docs/extracted/城主指南2024/4.创建冒险/冒险奖励.md) 换算。

### I5.5 委托池补充节奏

每次时间推进（日/周/月）DM 在背景刷新委托池：

- 每日：1-2 个 easy 委托
- 每周：1 个 medium 委托
- 每月：1 个 hard 委托
- PC 声望达 X 解锁特定来源（贵族派单 / 政府密谕）

DM 用 [城主指南2024/4.创建冒险/](../../docs/extracted/城主指南2024/4.创建冒险/) 的"冒险冲突"、"冒险背景"模板生成。**不要批量造**——每次生成 2-3 个，玩家接走再补，避免无意义堆积。

### I5.6 经验与升级（XP 系统）

沙盒默认采用 **PHB 标准 XP 系统（2014/2024 经验阈值一致）**——委托完成累积 XP，达阈值升级。可在 `house-rules.md` 切换其他 mode（详见末尾）。

> ⚠️ **沙盒默认单人 PC**。XP 由 PC 独吃，**不分**给协力 NPC / 永久伴生 / 配偶等。下表所有 XP 给法以"PC 当前等级 L"为参数。

#### XP 来源与给法

**1. 委托完成（核心 XP 来源）**

| difficulty | 委托完成 XP |
|---|---|
| easy | 25 × L |
| medium | 75 × L |
| hard | 200 × L |
| deadly | 500 × L |

例：3 级 PC 完成一个 medium 委托 → 225 XP。

**2. 重大事件 XP**（DM 一次性给）

| 事件 | XP |
|---|---|
| 救一个 NPC 性命 | 50-200（按重要度） |
| 重要 NPC 关系突破（揭穿身份 / 化敌为友 / 求婚成功） | 100-300 |
| 短期野望达成（开店成功 / 加入公会 / 第一个家庭） | 200-500 |
| 中期野望达成（结婚 / 立业完成 / 完成大冒险） | 500-1500 |
| 长期野望达成（领地建立 / 改变政局 / 史诗成就） | 2000-5000 |

DM 在桌末 Edit `players/<名>.md` 时加 XP，并在 `sessions/session-NN.md` 记录原因（让玩家知道 XP 怎么来的）。

**3. 不给 XP 的活动**
- 普通 downtime（日常训练 / 经商例行 / 拜访 NPC 闲聊）—— 避免刷级
- 失败的委托（部分 XP DM 酌情，最多 25%）
- 短休 / 长休 / 一般生活段

#### 协力 NPC / 同伴的 XP 处理

沙盒玩家身边可能有多种"非 PC"角色：

- **永久伴生**（驯兽 / 寻获魔宠 / 钢铁守护 / Beast Master 野兽魂魄）—— 角色卡 companion 形态
- **雇佣同伴**（按塔莎《协力者》数据块，月付薪水）
- **临时同行 NPC**（特定委托才带的协力 NPC）
- **配偶 / 家人**（达成婚姻后偶尔参与冒险）

**统一规则：PC 独吃 XP，协力 NPC 不分**。理由：单人沙盒长期跑，分 XP 会让 PC 升级速度被腰斩；且协力 NPC 主要服务剧情，不需要数值演进。

**等级处理**：

| 同伴类型 | 等级如何变化 |
|---|---|
| 永久伴生（魔宠 / 钢铁守护 等） | **自动跟随 PC**（PHB24 协力者数据块按 PC 等级缩放） |
| 雇佣同伴（专家 / 武者 / 施法者） | **按薪水档位写死**，不随 PC 升级；要升级需 PC 出钱训练 + in-game 时间 |
| 临时同行 NPC | DM 即兴定，单次委托用完即弃，不追踪 |
| 配偶 / 家人（非冒险者） | 不参与等级体系（平民身份） |
| 配偶 / 家人（冒险者出身，如退役游侠） | 在 NPC 档案里固定，不随 PC 同步升 |

**例外**：玩家明确"X 是我的死党 / 战友" + 长期固定同行（≥10 个 session 一起跑），DM 可让其慢于 PC 升级（差 1 级），用作戏剧张力——同伴成长滞后是好故事的张力来源（"我已经是 8 级冒险者，而你还在为 6 级的怪挣扎"）。

权威表见 [城主指南2024/2.运作游戏/角色升级.md](../../docs/extracted/城主指南2024/2.运作游戏/角色升级.md)。

#### 升级时机

XP 跨过阈值时**不立即升级**——按 PHB24 规则：

1. DM 提示玩家："你感觉这段时间的历练正在沉淀——下次长休完成后你将晋升到 N 级。"（in-fiction 表达，**不直接说"恭喜你升级了"这种 OOC**）
2. **下次长休完成时**生效（沙盒里就是"今晚睡一觉" / 下次日推进时）
3. 玩家在 HTML 角色卡上更新（HP / 法术位 / 新特性 / 新法术 / 子职选择 / 专长），完成后让 Claude 跑工作流 B' 重新生成 HTML，并 Edit `players/<名>.md` 同步

#### 节奏参考（单人沙盒，按每周 ~1 委托估算）

| 升级跨度 | 大约要做 | in-game 时间 |
|---|---|---|
| 1→2 | 4 个 medium 或 2 个 hard | 1-2 个月 |
| 2→3 | 8 个 medium 或 3 个 hard | 2-3 个月 |
| 3→4 | 12 个 medium 或 6 个 hard + 1 个短期野望 | 3-6 个月 |
| 5→6 | 50 个 medium 或 18 个 hard + 几个中期事件 | 1-1.5 年 |
| 10→11 | 100+ medium 或 35+ hard 或多个野望节点 | 2-4 年 |
| 15→16 | 海量委托或单一长期野望（统治领地） | 5-10 年 |

中高级升级**应该**慢——这是 5e 设计意图。沙盒的最大魅力之一就是让 PC 真的"在游戏世界活了 N 年"——年龄推进 / 婚姻 / 孩子成长 / 政治格局变化都跟着发生，升级慢反而是好事。

#### `house-rules.md` 配置

```yaml
xp_mode: standard       # standard / milestone / hybrid
```

- **standard**（默认）：上述 XP 系统
- **milestone**：不算 XP，DM 在以下时机统一升级——完成 3 个 medium → 升 2 级；完成 5 个 medium 或 2 个 hard → 升 3 级；达成短期野望 → +1 级；达成中期野望 → +2 级；达成长期野望 → +3 级
- **hybrid**：委托给"成长点"（折半 XP），具体升级时机由 DM 在野望达成时决定（兼顾透明感与节奏控制）

玩家随时可在桌上声明切换，DM Edit `house-rules.md` 即可。

### I5.7 任务演化机制

委托不是开桌时一锤定音的静态合同——玩家的行动（审讯 / 调查 / 选择 / 通过特定检定）可以**改变任务目标 / 升级难度 / 调整奖励 / 派生新委托**。这是沙盒区别于模组的核心张力之一：你接的不是"抓小偷"这个**任务**，而是"被偷东西"这个**事件**——它会顺着玩家的探询长出新的枝桠。

#### I5.7.1 何时触发演化判定

DM 在 §I5.3 步骤 3.5 主动检查以下几类触发：

1. **审讯 / 对话揭示**：玩家俘获 NPC 后追问动机 / 幕后 / 共犯，且通过 DC（说服 / 洞察 / 威吓）
2. **调查取证揭示**：玩家在现场翻找文书 / 信件 / 痕迹，且通过 DC（调查 / 侦察 / 奥秘 / 历史）
3. **观察识破伪装**：玩家发现 NPC 身份与公开描述不符（伪装的贵族 / 假死的死者 / 假冒的发布者）
4. **道德选择岔路**：玩家选择放过 / 招募 / 处决目标，每条路径开新支线
5. **第三方介入**：调查过程中遇到另一派系（焰拳 / 暗影盗贼 / 烛堡）撞上同一线索，触发竞争 / 合作支线
6. **预埋分支命中**：`secret` 段写明的"潜在演化分支"被玩家行动触发

**不该触发演化**的场景（避免泛滥）：
- 玩家只是闲聊几句但没问关键问题
- 没有任何检定支撑的"我猜……"
- 玩家完成原目标后没有任何后续探询行为（任务就该 `completed` 收尾，不必硬加阴谋）

#### I5.7.2 两种演化姿态

**追加型（常见）**——原目标完成，新目标承接：

适用：玩家完成原委托后通过审讯/调查发现更大事件。原 reward 仍然发放，新事件以子委托形式承接。

```yaml
# 原 COM-001 状态
id: COM-001
title: 寻找走失的猫
type: 寻物
status: completed                      # 原目标已完成
reward:
  base: 10 GP + 玛丽埃塔的好感         # 已发放
  pivot: null                          # 无 pivot；新奖励挂到子委托
chain:
  parent: null
  spawned: [COM-005]                   # 派生

# 子委托 COM-005
id: COM-005
title: 调查法师学徒的禁咒研究
type: 调查
status: open
reward: 100 GP + 烛堡好感 + 1 本古籍副本
difficulty: medium                     # 难度独立设定，不继承父任务
chain:
  parent: COM-001                      # 链接到父委托
  spawned: []
```

**替换型（rare）**——原目标作废，事件主线转向：

适用：玩家中途揭穿"原任务发布者本身就是反派"，原目标变得无意义。原 reward 不发，整个委托 status 转 `evolved`，主线推到子委托。

```yaml
id: COM-001
title: 寻找走失的猫  →  揭穿玛丽埃塔的伪装
type: 寻物 → 调查
status: evolved                        # 原目标作废
reward:
  base: null                           # 不发放（玛丽埃塔是假身份，原奖励就是个饵）
  pivot: 见 COM-005
chain:
  parent: null
  spawned: [COM-005]
revealed_branches:
  - date: 风岚月 第 5 日
    trigger: 玩家审讯学徒 DC 18 说服通过 + 调查银项圈 DC 14
    revealed: 玛丽埃塔本名是叛逃的烛堡学者，"猫"实为偷出的活体魔法实验体
    spawned: COM-005
```

#### I5.7.3 DM 演化操作清单（5 步）

玩家行动触发演化时，DM 当场执行：

1. **判定 + 决定姿态**：追加型还是替换型？多数情况下选追加型（保留玩家原 reward，避免"我做了半天白做"的挫败感）。
2. **写演进日志**：在原委托档案 `## 演进日志` 段追加一行散文：
   ```
   风岚月 第 5 日：玩家审讯学徒（DC 18 说服）+ 调查银项圈（DC 14 调查）→ 揭示玛丽埃塔
   实为叛逃烛堡学者，"猫"是魔法实验体 → 派生 COM-005
   ```
3. **填 `revealed_branches`**：把这次触发结构化记录到 frontmatter（date / trigger / revealed / spawned）
4. **新建子委托**：在 `active.md` 追加 COM-XXX，frontmatter 含 `chain.parent: <原 id>`，难度/奖励独立按 §I5.4 重新定，不继承父任务
5. **回填原委托**：
   - 更新 `chain.spawned: [COM-XXX]`
   - status 按姿态改 `completed`（追加型）或 `evolved`（替换型）
   - reward 从字符串改为对象 `{base, pivot}` 分段记录
   - type 字段可写为 "原类型 → 新类型" 显示转向

#### I5.7.4 演化奖励的设计原则

- **追加型**：base 按原 difficulty 给（不打折扣），pivot 挂在子委托独立计算
- **替换型**：base 通常不发；但 DM 可酌情给"调查辛苦费"（原报酬的 25-50%）作为安抚
- **不能"演化通胀"**：一个 easy 委托派生出三层后变成 deadly + 5000 GP 的不合理升级要避免；每层至多升一档难度，且新难度要有相应场景支撑（不能 RP 一下就拿到 hard 奖励）
- **XP 各自结算**（§I5.6）：父委托完成给 `25 × L`（easy），子委托完成另算 `75 × L`（medium）—— 两笔 XP 都给，玩家因"深挖"得到合理回报

#### I5.7.5 演化链的归档

- 父委托 `completed` / `evolved` 时归档到 `commissions/completed/NNN_<名>.md`，**保留 `chain.spawned` 字段**
- 子委托完成时同样归档，`chain.parent` 保留 —— 这样翻 `completed/` 文件夹时能复原整条链条
- 桌末 `sessions/session-NN.md` 提到任务演化时，简单写"COM-001 → COM-005（学徒禁咒线）" 即可，详情看委托档案

#### I5.7.6 预埋 vs 即兴

DM 出委托时**可以**（但不强制）在 `secret` 段预埋 1-2 条潜在演化分支：

```yaml
secret: |
  🔒 这只猫被一个练习召唤术的法师学徒偷去做实验。
  🔒 潜在演化分支 A：玩家审讯学徒 DC 12+ → 揭示导师禁咒研究 → 派生调查委托
  🔒 潜在演化分支 B：玩家放过学徒 + 收他为线人 → 学徒之后可作情报源（写入 relationships/）
  🔒 潜在演化分支 C：玩家把学徒交给烛堡 → 烛堡声望 +10，但学徒被处决，导师警觉销毁证据（COM 永久关闭）
```

预埋的好处：DM 跑团时不必当场即兴；缺点：可能限制即兴爆发。**MVP 建议**：只给 hard 委托和主线相关委托预埋；easy/medium 让 DM 跟着玩家行动即兴长出来。

#### I5.7.7 玩家视角

**绝不**告诉玩家"这个委托会演化"——演化感来自玩家**主动探询**后发现的意外。任何"提示玩家有更深内容"的元话语都破坏沙盒探索张力（违反 G4 信息揭示边界）。子委托出现时，让它**自然**地通过 NPC 委托 / 告示板 / 偶遇出现，不点明"这是 COM-001 的延伸"。

玩家若在桌末问"那只猫的事和烛堡禁咒有关吗"，DM 可承认；但不应在事件发生时就标注因果链。

## I6. 关系演进协议

### I6.1 关系阶段

每个 NPC 在 `sandbox/relationships/<NPC>.md` 维护好感度（-100 到 +100）和阶段：

| 阶段 | 阈值 | 触发事件 |
|---|---|---|
| 仇敌 | -80 ~ -100 | 攻击 / 陷害玩家 |
| 厌恶 | -50 ~ -79 | 言语攻击 / 不帮忙 |
| 冷淡 | -20 ~ -49 | 例行公事 / 不主动 |
| **陌生**（起始） | 0 ~ ±19 | 默认 |
| 熟识 | 20 ~ 39 | 主动打招呼 / 偶尔闲聊 |
| 友好 | 40 ~ 59 | 主动帮忙 / 分享信息 |
| 亲密 | 60 ~ 79 | 透露秘密 / 危难相助 |
| **暧昧 / 恋人**（恋爱线开启时） | 80 ~ 89 | 见 §I6.4 恋爱线 |
| **配偶**（婚姻后） | 90 ~ 100 | 共同生活 / 家庭事件 |

阶段是平台期，到阈值时**触发对应事件**，事件成功后稳定到新阶段。

### I6.2 好感度变动来源

| 行动 | 变化 |
|---|---|
| 普通对话（每段最多一次） | +1 |
| 满足 NPC 需求 / 完成 NPC 委托 | +5 ~ +20（按重要度） |
| 送对的礼物（投洞察查兴趣 → 准确） | +5 ~ +15 |
| 送错的礼物（敷衍） | 0 ~ +2 |
| 共同冒险 / 救命 | +10 ~ +30 |
| 公开支持 / 信任 | +5 ~ +15 |
| 背叛 / 背诺 | -20 ~ -50 |
| 触犯禁忌（信仰 / 家族） | -10 ~ -40 |
| 不联系（每月） | -5（同居/亲属除外） |

DM 每次触发后 Edit `relationships/<NPC>.md` 的"互动历史"段，追加一行：`第 N 日 - 事件 - 好感±N → 当前 X`。

### I6.3 阶段触发事件（自动剧情节点）

跨阈值时 DM **必须**触发对应剧情节点：

- 跨 20（陌生→熟识）：NPC 主动打招呼 / 称呼变化（"喂"→姓名）
- 跨 40（熟识→友好）：NPC 邀请玩家参加私人活动（吃饭 / 出游 / 节日）
- 跨 60（友好→亲密）：NPC 透露一个秘密 / 求一次重要帮助（"信任测试"）
- 跨 80（亲密→暧昧 或 战友 / 死党）：见 §I6.4
- 跨 90（暧昧→配偶 或 血盟兄弟）：婚礼 / 血盟仪式

### I6.4 恋爱 / 婚姻线（按 DM 风格）

**开启条件**：DM 风格 = `粉红恋爱向` 或玩家明确表示对某 NPC 有意 + NPC 设定可恋爱（未在 `dm-only/npcs-secrets/<NPC>.md` 中标 `romance_locked: true`）。

风格基调由 [profiles/dm-styles/粉红恋爱向.md](profiles/dm-styles/粉红恋爱向.md) 决定。其他风格（治愈日常 / 黑暗）也可有恋爱线，但表达方式不同。

**恋爱进度**：

1. **暧昧期**（好感 60~79）：NPC 表现微妙的关注 / 借口找玩家
2. **告白事件**（好感 ≥ 80）：触发"告白时刻"——可能 NPC 先表白，也可能 DM 让玩家有机会先表白
3. **恋人期**（好感 80~89）：可以同住 / 公开 / 共冒险（NPC 加入 PC 同行——按 [塔莎/协力者/](../../docs/extracted/塔莎的万事坩埚/协力者/) 简化为协力者数据块）
4. **求婚 + 婚礼**（好感 ≥ 90）：玩家或 NPC 求婚 → 婚礼场景（仪式按选择神祇 / 风俗）→ 关系阶段更新为"配偶" → Write `family.md` 添加配偶
5. **婚后**：见 §I8 家族系统

**关键纪律**（呼应 I0 风味是叠加层原则）：

- 恋爱发展**不替代**风险：恋人可被绑架 / 被诅咒 / 卷入政治；婚礼可被搅局；配偶可背叛或牺牲
- 同时谈多个 NPC：每个 NPC 独立追踪好感，但若两个 NPC 同时进入恋人期，DM 必须触发"撞见 / 嫉妒"事件让玩家面对选择
- 强暧昧 ≠ 必然恋爱：NPC 拒绝是合法剧情（按 NPC 性格档案 / DM 风格）

### I6.5 关系档案 schema

```yaml
---
name: 艾莉安娜
race: 半精灵
public_role: 烛堡图书馆助理馆员
affinity: 65          # 好感度（-100~+100）
stage: 友好           # 当前阶段
romance: false        # 是否进入恋爱线
known_since: 风岚月 第 2 日

gift_history: [{ date: 风岚月 第 10 日, item: 银发夹, value: 5GP, effect: +8 }]
secrets_revealed: []  # 已揭示给玩家的 NPC 秘密
---

## 公开档案
（玩家视角已知）...

## 兴趣 / 厌恶（DM 查表用）
喜欢：古籍 / 月光石 / 安静散步
厌恶：油腻食物 / 喧哗 / 公开场合

## 互动历史（追加式）

- 风岚月 5 日 - 接她"找一本走失的书"小委托完成 → +20 (25)
- 风岚月 10 日 - 送银发夹 → +8 (33)
```

### I6.6 NPC 秘密档案（私密版）

`dm-only/npcs-secrets/<NPC>.md` 同 G 工作流：保留 NPC 真实身份 / 动机 / 隐藏剧情 / 揭示条件。在沙盒里**额外加**：

```yaml
romance_locked: false       # true 则不可恋爱
romance_partner_pref: any   # any / 同种族 / 男 / 女 / 同性 / 异性
betrayal_trigger: |
  若玩家曾对她说谎且 affinity 跌至 40 以下，触发离开城镇事件
```

## I7. 资产经营

### I7.1 起始资产

按角色等级 + 背景 + DMG24 起始财富。沙盒玩家通常**比模组玩家更穷**（沙盒长期玩，前期穷一点才有挣钱动力）。

### I7.2 房产

`sandbox/assets/properties.md`：

```yaml
---
properties:
  - name: 下城小阁楼
    type: 公寓                # 公寓/单层屋/独栋/庄园/塔楼/堡垒
    location: 博德之门-下城-柳树街
    purchase_date: 风岚月 第 15 日
    purchase_price: 200 GP
    monthly_cost: 5 GP        # 维护费（含水电柴薪等）
    security: 普通            # 简陋/普通/良好/重防/魔法防护
    capacity: 2 人
    notes: 二楼小屋，邻居是个药剂师
---
```

价格参考 PHB24 + DMG24 + Xanathar's。简表：

| 类型 | 起价 | 月维护 |
|---|---|---|
| 房间出租 | — | 0.5-2 GP / 月 |
| 公寓 | 100-500 GP | 2-10 GP |
| 单层屋 | 500-2000 GP | 10-50 GP |
| 独栋 | 2000-5000 GP | 50-100 GP |
| 庄园 | 5000-25000 GP | 100-500 GP |
| 塔楼 / 堡垒 | 25000+ GP | 500+ GP |

### I7.3 业务（产业）

`sandbox/assets/business.md`：

```yaml
---
businesses:
  - name: 灯塔之眼药材店
    type: 商店                # 酒馆/商店/作坊/医馆/赌坊/旅店
    location: 博德之门-下城
    purchase_date: 风岚月 第 30 日
    purchase_price: 500 GP
    monthly_revenue_base: 80 GP
    monthly_cost: 50 GP       # 进货+雇员+维护
    employees: [{ name: 多丽丝, role: 学徒, salary: 5GP/月 }]
    reputation: 普通          # 烂/差/普通/好/极佳
    notes: 主营治疗药水原料
---
```

**月底自动结算**：营业额 = base × roll(0.6-1.4)，roll 由市场事件 + 月内玩家投入决定：

- 玩家每月亲自经营至少 1 段时间 → +0.1
- 雇佣高级店员（玩家做检定面试） → +0.2
- 突发竞争 / 治安事件 → -0.2 ~ -0.4
- 触发 DMG24 [阵营.md](../../docs/extracted/城主指南2024/3.地下城主工具箱/阵营.md) 阵营关系 → 加值或减值

参考资料：[艾奎兹玄/发展公司/休整期/](../../docs/extracted/艾奎兹玄有限责任公司/发展公司/休整期/) 9 个细分行动，按需借用机制。

### I7.4 库存 / 生活水准

`sandbox/assets/household.md`：

```yaml
---
living_standard: comfortable   # squalid/poor/modest/comfortable/wealthy/aristocratic
monthly_household: 30 GP       # 按 PHB24 表
food_supply: 30 日             # 储备天数
luxuries: []                   # 收藏品 / 享受
servants: []                   # 仆人
---
```

生活水准影响：
- 社交圈层（高水准开放贵族派对）
- NPC 第一印象（衣着检定加值）
- 健康（squalid / poor 时月度体豁 DC 10，失败患病）
- 法术研究效率（wealthy+ 加成）

## I8. 家族系统（5e 无原生规则，轻量自创）

### I8.1 设计原则

家族系统是**叙事工具**，不是模拟器。所有数值服务于剧情张力，不追求生理准确。

### I8.2 婚姻

进入"配偶"阶段（好感 ≥ 90 + 完成婚礼仪式）后，配偶写入 `sandbox/family.md`：

```yaml
---
spouse:
  name: 艾莉安娜
  race: 半精灵
  marriage_date: 风岚月 第 90 日
  ceremony_type: 烛堡神殿（密丽尔）
  living_with_player: true
  affinity: 95
status: married
children: []
parents: []                    # 玩家或配偶的父母（按需）
siblings: []
in_laws: []
---
```

配偶身份**追加效果**：
- 自动每月互动（不衰减）
- 共同账户（配偶资产合并到玩家档案）
- 玩家不在家时配偶可独立行动（DM 即兴生成日常）
- 危机事件：配偶可被绑架 / 误会 / 怀孕等 → 触发剧情

### I8.3 怀孕

任一方提议且双方同意（玩家声明 + DM 与配偶 in-character 互动） → 投体豁 DC 10（双人 RP 后默认成功，可 house-rule 调整）→ 标记怀孕日期。

```yaml
pregnancy:
  conceived_date: 风岚月 第 100 日
  due_date: 风岚月 第 370 日   # 9 个月后
  carrier: spouse              # spouse / player（如玩家是女性角色）
  complications: []            # 怀孕并发症
```

**孕期月度事件**（按月推进时触发）：
- 第 1-3 月：孕吐 / 检查 / 选名
- 第 4-6 月：胎动 / 知道性别 / 添置物品
- 第 7-9 月：行动不便 / 配偶不能冒险 / 准备产房
- 临产：玩家在场 → 全程剧情；不在场 → 仆人通知

**生产**：产妇体豁 DC 12（孕期事件触发的并发症加 DC）→ 成功生产；失败启用 DM 风格决定后果（治愈日常风格：调低 DC 或允许"信仰加值"；黑暗风格：可能难产）。

## I12. 与工作流 G 的接口（如何混合）

### I12.1 沙盒嵌套模组

玩家在沙盒中可以"接一个完整的模组冒险"：

1. DM 在委托池中投放一个"高 stakes 委托"，链接到 `docs/modules/<某模组>/`
2. 玩家接受 → 切到 G 工作流跑完整模组
3. 模组结束 → 回到沙盒，把模组的成果 / 损失同步到沙盒档案（金钱 / 物品 / 伤亡 / 揭示的世界信息）

示例：玩家沙盒中接到"风骸岛的修道院求援"委托 → 切 G 跑风骸岛之龙 → 完成后回博德之门继续沙盒生活，茹娜拉成为可联系 NPC（写入 `relationships/茹娜拉.md`）。

### I12.2 模组转沙盒

玩家完成一个模组后，DM 可建议"留在 X 地开沙盒"——把模组的 NPC / 地点继承到沙盒档案：

1. Read 模组 NPC 速查 → 把活下来的 NPC 复制到 `sandbox/relationships/<X>.md`（初始 affinity = 跑团互动累积）
2. Read 模组地点 → 把可继续访问的地点复制到 `sandbox/locations/<X>.md`
3. 调整 progress.md：从"模组章节"切换为"野望追踪"
4. 在 README.md 改 `mode: module` → `mode: sandbox`，加 `previous_module: <模组名>`

### I12.3 沙盒中的战斗

沙盒里发生战斗（接的委托 / 突发遭遇 / 家族危机）一律切 G3 战斗管理：

- Write `combat/active.md`
- 战斗结束归档到 `combat/history/NNN_*.md`
- 战斗结果同步到 `players/<名>.md`（HP / 法术位 / 装备）+ `treasury.md`（消耗 / 缴获）

战斗叙事仍遵循"风味是叠加层"——沙盒里的战斗**不是**RP 装饰，是真实的战术决策（PC 可能死，NPC 可能死）。

### I12.4 桌末记录（在 G6 基础上扩展）

每桌结束依次同步：

1. Write `sessions/session-NN.md`（同 G6.1）
2. Edit `players/<*>.md`（同 G6.2）
3. Edit `progress.md`（沙盒里：野望追踪 + 已达成里程碑）
4. Edit `world-state.md`（同 G6.4）
5. Edit `dm-only/dm-notes.md` + `dm-only/sandbox-notes.md` ⭐
6. Edit `npcs/<X>.md`（同 G6.6）
7. **沙盒专属**：Edit `sandbox/calendar.md` 推进日期 + 更新未结事件 ⭐
8. **沙盒专属**：Edit `sandbox/relationships/<本桌互动>.md` 同步好感度 ⭐
9. **沙盒专属**：Edit `sandbox/commissions/active.md` 同步委托状态 ⭐
10. **沙盒专属**：Edit `sandbox/assets/treasury.md` 同步金钱流水 ⭐
11. **沙盒专属**：若有家庭变化，Edit `sandbox/family.md` ⭐
12. Edit 战役 `README.md` frontmatter：`session_count += 1`，`in_game_start` 不变但要在 calendar.md 加新条目
